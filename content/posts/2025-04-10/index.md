# 面試心得分享｜從離職到新工作上岸的歷程

## 🧳 離職初衷與職涯規劃

離開了一份待了四年多的工作—說長不長、說短不短，但那是一段我傾注熱情與心血的時光。  
這段期間，我從 RD 做到 Team Leader，甚至跨組織帶領團隊開發國際版產品，協助推動內外部成本改善、對接匯報、負責個人績效考核與年度計畫制定、也處理過 PIP（績效輔導）。這一路上，我真的學到很多。

然而，因為某些轉捩點的出現，我知道，是時候準備下車了。

離職前，我曾短暫加入一家接案公司，不到一個月。那段時間讓我重新面對了新的技術堆疊、開發流程的落差，也更清楚知道自己要的是什麼、不要的是什麼。  
於是我與公司溝通後，決定「裸辭」。很感謝當時的主管，給我高度的尊重與支持，甚至願意幫我媒合其他資源——我始終相信，**在職場上最重要的資產就是「個人口碑」。**

你問我，有沒有後悔裸辭？  
完全沒有。這是我職涯中非常關鍵的一步，也是一個需要勇氣的決定。

離職後，我出國旅行了一趟，好好地整理了自己。  
我開始重新思考接下來的方向：我希望加入一個能實際參與技術決策、有影響力的團隊，繼續發揮我的系統設計與跨部門協作經驗。  
同時，我也給自己設下一個目標——**挑戰百萬年薪。**

這段求職旅程花了大約 2~4 個月，過程中有焦慮、有期待，但更重要的是：我又更認識自己了一次。而我也真的做到了。

## 📮 投遞履歷與收到面試邀請

更新完履歷後，我將它上架到 104、CakeResume 等平台，並根據不同公司與職缺，調整自傳與作品集（Side Project）內容。

這次也趁機把自己過去累積的技術做一次回顧與實戰：  
我重新摸熟了一輪 **Golang** 和 **Node.js**，也開始接觸 **OpenAI**、**Prompt 工程**，嘗試將一些 AI 工具應用到生活中。  
把原本只是 POC 的構想，逐步實作成 Side Project，也讓我在面試時更有信心展示自己。

大約兩週內，開始陸續收到不少邀約，其中不乏我特別感興趣的公司，包括：

- 🏢 **客立樂 QLiEER** - 後端工程師

- 🧘 **喬山健康科技 Johnson Health** - 主任工程師
- 🧠 **德鴻科技** - 技術副理

這些職缺的職責與我期望的發展路線高度契合，每間公司的文化、技術堆疊與成長空間也都讓我非常期待。  
於是我開始密集安排面試，正式進入下一階段的挑戰。

## ⏳ 面試流程真的不簡單

光是讓履歷被看見，其實就已經是第一道門檻。  
即使順利拿到面試邀請，前置流程也很花時間，像是：

- 面試前要來回確認面談時間
- 有些公司會先給技術測驗或問卷作答
- 每個職位通常都會有 **3～4 關** 的面試（HR / 技術面 / 部門主管 / 文化面試 等）
- 若進入敘薪與 offer 階段，通常還會再來一輪互動確認

所以一份完整的求職流程，從投遞、筆試、面試到最終談薪
**動輒耗時 1～2 個月以上是常態**。  
而常見的面試流程會包含以下幾類：

- **HR Screening**：人資初步聯繫，確認動機、履歷背景與待遇期望
- **Technical Interview**：技術面談，可能包含 live coding、系統設計或過往專案探討
- **Hiring Manager Interview**：部門主管面談，聚焦在未來角色、合作方式、成長性
- **Behavioral Interview (BQ)**：文化與人格特質面談，常用 **STAR 法則** 來回答問題（Situation, Task, Action, Result）
- **Offer Discussion / Salary Negotiation**：最終敘薪與確認入職條件

有些公司還會再多加一輪 **Take-home Assignment** 或 **Peer Interview（同儕面談）**，進一步了解協作方式或技術深度。

整體來說，每一輪都是一場腦力與耐力的挑戰，但也幫助我更認識自己真正想加入什麼樣的團隊與文化。

### 📝 面試過程（第一階段）

- **時間**：3/26
- **前置作業**：DISC 性格測驗

當天到了公司三樓後，由兩位 RD 接待，一位是後端工程師、一位是技術主管。整體面談包含：

#### 🔍 技術題與經驗探討（約 20 分鐘） 大約 10~ 15 題

1. 為什麼選擇離職？
2. 前一份工作內容？使用過哪些技術與工具？
3. 擔任 Team Leader 時的角色與分工比例？
4. 過去在 API 設計、雲端架構、自動擴容、CI/CD、Docker 的經驗？
5. 高併發處理經驗？如何解決 race condition、鎖的應用（悲觀/樂觀/原子鎖）？
6. 面對資料量大與 SQL 效能問題時的分析與優化方式？GCP/AWS 的應用？
7. 微服務設計與 K8s 實戰經驗？

#### 🤝 行為面試（Behavioral Questions）（約 20 分鐘） 大約 8~ 10 題

1. 團隊內部發生爭執時你如何處理？
2. 跨部門溝通遇到阻礙時如何解決？
3. 曾參與國際版產品開發，有遇到哪些困難？如何克服？
4. 曾導入的開發流程是什麼？曾遇到的瓶頸與對應方案？
5. 你目前的短期／中期職涯目標是什麼？

### 💻 Side Project Demo（約 20 分鐘）

我展示了兩個用於面試的 side project，皆使用 **TypeScript / Node.js / NestJS** 為主軸，結合測試與設計模式：

#### 🔹 專案一：設計模式實作 + CI/CD
- 使用 TypeScript 搭配 Jest、ESLint，結合 GitHub Actions 做 CI
- 將 `singleton`、`factory`、`strategy`、`proxy` 等常見設計模式模組化
- 串接 Codecov 顯示測試覆蓋率報表

#### 🔹 專案二：通知微服務系統(延伸專案一的主軸)
- 使用 NestJS + Fastify 架構整體專案
- 整合 RabbitMQ、Redis、Swagger API、Unit Test、Docker 化
- 使用 `factory` 實作多型通知（Email / SMS）
- 使用 `proxy` 快取通知列表，並用 `k6.js` 比較有無快取的效能差異（RPS、p90、p95 latency、error rate）

主管與工程師聽完後也針對其中的實作與設計進行了一些簡單討論。

一開始照例先自我介紹，簡單說明我的背景與轉職動機。  
接著，就進入重頭戲——這一場面試長達 **40～50 分鐘**，幾乎都是由 CEO 與 COO 輪番提問與深入對談。

#### 🎯 面試內容重點比重：

- **約 5%**：針對過往的技術經驗提問
- **約 95%**：**Behavioral Questions（行為情境式問題）**，深入探討我在「酷必」擔任主管期間的實務經驗與決策方式：

內容圍繞著以下幾個方向展開：

- 如果是你，你會怎麼導入這項技術？
- 如何在有限資源下做出成本優化？
- 架構重構的時機點與風險怎麼評估？
- 作為主管，你如何協助團隊提升產出效率？

這場面試其實比較像一場深度對談，兩位高層一來一往，丟出一連串針對實戰經驗的延伸問題。

我幾乎是無縫接軌地回答每一題——事後回想，真的非常推薦在這種面試中使用 **STAR 應答法**（情境 Situation → 任務 Task → 行動 Action → 結果 Result），可以幫助自己條理分明又真誠地說出經驗。

這是我此次面試中所接觸到規模最大、**同時也是全球百大企業（排名第 80）之一** 的公司。

其實早在四五年前，離開 BC 產業時就聽過「數數發」來台中設點的消息，當時就有前同事加入、並傳來不錯的回饋。這份職缺一直是我心中的理想清單之一。

### 🎯 如何準備 SA 面試？

說真的，在這麼多競爭者中被看到、甚至錄取，我真的感到非常榮幸。為了爭取這個機會，我做了非常多的準備：

- **每天進行 AI 模擬面試**，針對 SA/系統架構師常見問題反覆練習
- 訓練自己在需求確認、設計收斂、系統規劃、跨組織溝通、成本評估等方面的答題能力
- 研究金融業的開發文化、法規限制與開發流程

我也特別針對自我介紹與履歷重新調整內容，讓經驗與職務更加對齊。

回顧自己第一份工作的經驗，真的非常有幫助——我曾經獨立應對客戶，負責需求訪談、現場審核、教育訓練等工作，對於需求從 0 到 1、從落地到上雲的全流程相當熟悉。

此外，在 BC 時期我就主動承擔 SA/SD 的責任，而在酷必擔任 Leader 期間，更是有大量的跨部門溝通、需求收斂、系統設計與報價建議經驗。  
所以這次面試能夠順利，我真的想說一句：**「感謝過去的自己。」**

### 🖥️ 第一面試（3/20｜視訊）
### 🧪 面試之間的作業

- **需完成線上測試(邏輯、數理、英文)**
- 由一位 SA 與一位 Hiring 主管進行
- 約 一小時 ，密集問答，腦力激盪

從自我介紹開始，便延伸出許多針對我過往經歷的 **Behavioral Questions（情境式問題）**，像是：

- 「如果你在這個情境中，會怎麼設計解法？」
- 「面對需求不斷變動，你會如何收斂？」
- 「過去有跨組織協作經驗嗎？怎麼做？」

這場面試讓我更有信心，覺得自己應該會進到下一階段的「聯合面試」（通常由高層與未來合作單位一同參與）。

這份職位的主要內容，是負責 **面向東南亞市場的系統開發與分析設計**，並在**確保法規合規性（Regulatory Compliance）** 的前提下，導入合適的技術與系統架構，推動數位轉型。

具體職責包括：

- ✅ **跨國系統開發與協作**：參與海外系統之開發與維運，與 PM、Developer、QA、外部 Vendor 密切合作，協調需求與進度，確保產品品質與交付節奏。

- ✅ **需求訪談與業務分析**：與內部業務單位進行訪談，挖掘與歸納需求，進行商業流程分析與系統功能分析，並評估功能模組的可重用性、實作複雜度與可維護性。

- ✅ **系統設計與規格建立**：設計滿足功能性（Functional）與非功能性（Non-Functional）需求的系統架構，撰寫技術規格書，並導入高可用性、高效能的解決方案。

- ✅ **跨部門溝通與技術推進**：與 PM 協作規劃專案進度，並與內外部 IT 團隊進行技術整合與持續溝通。實作過程中，與 Developer、QA 合作進行問題排查、效能優化、測試案例設計與系統驗證，確保產品達到上線標準。

這個職位不只是技術導入者，更是橫跨 **需求、分析、設計、交付與落地** 的橋樑，需要具備清晰邏輯、溝通協調力與跨文化合作能力。

### 🧑‍💼 聯合面試（3/27｜視訊）

這一關是最緊張的。

- 出席者：4～5 位高層主管 + 1 位 HR
- 每位主管各問 2～5 題問題
- 整場由一位高層主導掌握節奏與重點

其中一位主管的提問方式屬於「壓力測試型」，不斷挑戰、質疑我的過往經驗與能力，甚至提出許多反駁與打壓式的問題。

我理解這是一種測試，看你能不能在高壓情境下穩定表現、邏輯清楚。  
在壓力下，我也嘗試透過其他角度補充說明，用邏輯說服他們我有能力勝任職位。

坦白說，前 20～30 分鐘真的非常有壓迫感，是我這次面試過程中節奏最不好掌控的一場。
以往我會透過 **穩定的語速**、刻意將 **聲音放低、語調平穩** 來傳達自信與專業感，並在適當時機主動發問或引導討論，藉此**掌握對談節奏、建立雙向溝通的主控權**。

但這次的節奏實在太快，加上連番追問與挑戰式的提問，讓我一開始較難站穩立場。不過我還是盡力維持邏輯清晰，逐題應對，也不斷提醒自己：**「穩住，不慌；只要回答真實的經驗，就一定說得出來。」**

雖然如此，我還是盡全力讓自己的回答維持有條理，靠著練習累積的反應速度與應變力，一題一題穩穩回答下來。  
整場約 1 小時，結束時我其實非常感謝自己——雖然過程中有些緊張，但我真心覺得每個回答我都已經做到最好。

### 💼 後續與敘薪流程

- 面試通過後，HR 在兩週內聯繫進入敘薪流程
- 會詢問你的 **薪資期待範圍**，並以「年薪制」為計算依據
- 根據考核等級，年終獎金約落在 **4～6 個月**
- 兩週內即收到 offer 🎉 (面試到 offer get 大概快 2 個月)
